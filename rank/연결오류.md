# 연결 오류 및 해결 방법

## 1. 프론트엔드 실행 오류 (Blank Screen)
### 현상
- 웹 페이지가 하얗게 나오고 아무것도 뜨지 않음.
- 브라우저 콘솔(F12)에 `Uncaught ReferenceError: createContext is not defined` 에러 발생.

### 원인
- `src/contexts/AuthContext.jsx` 파일에서 React 라이브러리의 필수 함수들(`createContext`, `useState` 등)을 import 하지 않아서 발생.

### 해결
- 해당 파일 상단에 `import React, { createContext, useState, useEffect, useContext } from 'react';` 구문 추가함.

## 2. API 400 Bad Request 오류
### 현상
- 랭킹 페이지 진입 시 `/ranking` API 호출이 실패하며 `400 Bad Request` 응답이 옴.
- 화면에 데이터가 표시되지 않음.

### 원인
- API Gateway Proxy Integration 사용 시, `event['body']`가 문자열(JSON String)로 전달되는데, 람다 코드에서 이를 처리하는 과정이 단순했음.
- 또는 `Content-Type` 헤더 처리가 API Gateway 설정과 맞지 않아 Body가 넘어오지 않았을 가능성도 있음(fallback 로직 취약).

### 해결
- **코드 수정 완료**: `d:\ai\aws_project\backend_lambda\ranking.py`
  - `event.get('body')`가 문자열인지 확인하고 안전하게 `json.loads()`를 수행하도록 로직 보강.
  - 파싱 실패 시에도 400 에러 대신 빈 페이로드(`{}`)로 처리하여 로직이 계속 수행되도록 수정.
- **조치 필요**: 수정된 `ranking.py` 파일을 AWS Lambda 콘솔에 다시 업로드(Deploy) 해야 함.
